# 数据清洗能力升级方案

## 已完成改进

### 1. 分段提取 + 跨章节上下文
每个章节独立调用 LLM，每段带文档标题 + 上级章节标题。前一章节的摘要（提取了哪些规则/内容概要）传递给下一章节，避免跨章节规则丢失前置条件。

### 2. Few-shot 示例
Prompt 中加入退款规则和权限管理两个高质量提取示例，稳定输出格式和质量。

### 3. Structured Output (tool_use)
使用 Anthropic tool_use 强制输出 schema，替代自由 JSON 解析。schema 定义了 rule_text、category、structured_logic、params、confidence 等字段。

### 4. 二次校验
提取完后再调一次 LLM，把提取结果和原文一起给它验证：
- 每条规则是否准确（不是幻觉）
- 是否完整（没有丢失关键条件）
- 是否有遗漏的规则（自动补充）

### 5. 提取后去重
用 embedding 相似度自动合并重复规则（阈值 > 0.95），保留置信度高的。

### 6. 跨文档规则合并
新提取的规则和已有规则比较（相似度 > 0.93），高度相似的自动关联来源文档而不是重复创建。

### 7. 置信度评分
每条规则带 confidence 分数（0-1），低置信度规则进入审核队列。

### 8. 人工审核入口
新增「规则审核」页面，展示所有 draft 状态规则，支持逐条通过/拒绝和批量通过。

### 9. 提取过程可观测性
新增 extraction_logs 表，记录每章节提取耗时、input/output tokens、规则数、错误信息。提供 `/api/analysis/extraction-logs/{prd_id}` 查询接口。
